Why the Admin portal exists

How it works

A simple admin workflow: verification → oversight → audit, designed for traceability.

01

Review verification requests

Open pending verifications, check identity/credentials, then approve or reject with a recorded reason.

02

Grant role-based access

Only verified accounts gain access to clinician/admin tools and sensitive platform areas.

03

Monitor case progression

Track cases moving to consensus and identify delays (e.g., waiting for quorum or missing reviews).

04

Audit and respond

Use logs to trace activity, investigate issues, and apply controlled security actions if needed.

FAQ

Quick answers for admin users.

What is the admin role for?

Governance and safety: verify accounts, oversee case progression, and review audit logs.

Can admins edit clinical results?

Admins manage access and oversight. Clinical consensus is generated through clinician reviews and system rules.

What should be logged?

Any sensitive action: approvals/rejections, role changes, security actions, and key oversight events.

How do verifications work?

Admins review identity and professional evidence, then approve or reject with a recorded reason.

What if a case is stuck on “Waiting for consensus”?

Check quorum status, missing reviews, and system errors. Use logs to identify where the flow stopped.
